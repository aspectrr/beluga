## Key Design Decisions

- `CreateSession` in ExtensionContext takes an `initialMessage` parameter (matches `orchestrator.HandleNewSession` signature which seeds the first user message)
- Docker client is `interface{}` in ExtensionContext to avoid requiring docker import for all extensions
- Extension `Start()` runs in goroutines so blocking extensions (connectors) don't block startup
- Extension `Stop()` runs in reverse registration order for proper dependency teardown
- System prompt assembly reads all `.md` files from `.beluga/prompts/` and appends them to SYSTEM.md content with headers
