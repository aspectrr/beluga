# Task 13-14-15: Session Store, Event Store, Tool Registry, Workspace Tools, Sandbox, Manager

## Files Created

### 1. internal/core/session/store.go

Session CRUD store using pgxpool.

- `Store` struct with `*pgxpool.Pool`
- `NewStore(pool) *Store`
- `Create(ctx, source, sourceID, metadata)` — inserts, returns with generated UUID + timestamps
- `Get(ctx, id)` — by ID, returns typed error on not found
- `GetBySource(ctx, source, sourceID)` — by source composite key
- `UpdateStatus(ctx, id, status)` — with rows-affected check
- `SetSandboxID(ctx, id, sandboxID)` / `ClearSandboxID(ctx, id)`
- `UpdateMetadata(ctx, id, metadata)` — JSONB merge with `||`
- `ListByStatus(ctx, status, limit, offset)` — ordered by updated_at DESC
- `ClearSource(ctx, id)` — archives source_id with `::archived::` + id suffix

### 2. internal/core/eventstore/store.go

Append-only event store with real-time watchers.

- `Store` struct with pool, `sync.RWMutex`, watchers map
- `Append(ctx, sessionID, eventType, data)` — transactional seq auto-increment, notifies watchers after commit
- `GetEvents(ctx, sessionID, afterSeq, limit)` — ordered seq ASC
- `GetLatest(ctx, sessionID)` — returns nil,nil if no events
- `GetEventsByType(ctx, sessionID, eventType, limit)` — newest first
- `StreamEvents(ctx, sessionID, afterSeq)` — buffered channel, sends existing then live events
- `notifyWatchers(evt)` — non-blocking send, drops if slow

### 3. internal/core/tools/registry.go

Tool registry and dispatch.

- `SandboxRunner` interface — `Exec`, `ReadFile`, `WriteFile` (satisfied by `workspace.Sandbox`)
- `Tool` interface — `Definition() ToolDef`, `Execute(ctx, args, ToolContext)`
- `ToolDef` struct — Name, Description, Parameters (JSON Schema)
- `ToolContext` struct — SessionID, Sandbox (SandboxRunner), EventStore
- `Registry` struct — thread-safe map with RWMutex
- `Register`, `Get`, `List`, `Execute` — standard CRUD + dispatch

### 4. internal/core/tools/workspace.go

Built-in workspace tools.

- `WorkspaceBashTool` — `workspace_bash`, runs bash -c in sandbox
- `WorkspaceReadFileTool` — `workspace_read_file`, reads from sandbox
- `WorkspaceWriteFileTool` — `workspace_edit_file`, writes to sandbox
- `RegisterWorkspaceTools(registry)` — registers all three
- All tools check `tctx.Sandbox != nil` before executing

### 5. internal/core/workspace/sandbox.go

Docker container wrapper.

- `SandboxConfig` — Image, CPULimit, MemoryLimit, NetworkMode, WorkDir
- `ExecResult` — Stdout, Stderr, ExitCode
- `Sandbox` struct — ID, SessionID, ContainerIP, CreatedAt, LastUsedAt, unexported cli
- `Exec(ctx, cmd, ...args)` — Docker exec with stdcopy demux
- `ReadFile(ctx, path)` — CopyFromContainer + tar reader, 10MB limit
- `WriteFile(ctx, path, content)` — tar archive + CopyToContainer

### 6. internal/core/workspace/manager.go

Docker sandbox lifecycle manager.

- `Manager` struct — cli, logger, sandboxes map, defaultImage, idleTimeout
- `ManagerConfig` — DockerHost, AgentImage, IdleTimeout, CPULimit, MemoryLimit
- `NewManager(cfg, logger)` — creates Docker client from env or config
- `Create(ctx, sessionID, cfg)` — container with sleep infinity, labels, no network default
- `Get(sessionID)` / `List()` — retrieval
- `Destroy(ctx, sessionID)` — force remove container
- `CleanupIdle(ctx)` — removes sandboxes idle > timeout
- `Close(ctx)` — destroys all, closes Docker client

## Interface Compatibility

`workspace.Sandbox` satisfies `tools.SandboxRunner`:

- `Exec` signature matches: `(ctx, cmd, ...args) (string, string, int, error)`
- `ReadFile` signature matches: `(ctx, path) (string, error)`
- `WriteFile` signature matches: `(ctx, path, content []byte) error`
